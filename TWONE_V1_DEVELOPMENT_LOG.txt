Twone 1.0 开发内容日志

版本：1.0
记录范围：本阶段从基础网站、会员系统、AI 助手，到每日分析正式数据源与自动发布链路的主要开发内容。


一、网站与基础容器搭建
1. 完成 Twone 网站基础项目搭建
2. 站点部署到 Vercel
3. 绑定正式域名 twone.xyz
4. 首页、申请页、助手页、后台页基础路由建立
5. 网站逐步从介绍页结构收口为交易产品结构


二、会员申请、审核与会员体系开发
1. 完成会员申请页与申请提交流程
2. 打通 Supabase 表单入库
3. 后台支持审核申请
4. 后台支持通过 / 拒绝操作
5. 支持会员状态维护
6. 支持免费体验 / Pro / VIP 区分
7. 完成会员入口和后台管理基础链路


三、登录、激活与密码链路开发
1. 普通用户支持密码优先登录
2. 支持首次设置密码 / 激活流程
3. 管理员后台支持单独登录入口
4. 排查并修复管理员登录过程中的密码链路问题
5. 增加管理员站内设置 / 重设密码能力
6. 降低管理员依赖 recovery 邮件跳转的问题


四、AI 助手方向收口与能力迭代
1. AI 助手从泛聊天收口到交易分析导向
2. 完成 BTC-first 的分析方向收口
3. 建立固定分析模板输出思路
4. 逐步增强分析风格：
   - 更短
   - 更硬
   - 更像交易员表达
5. 支持固定分析入口与会员能力区分
6. 后续继续迭代多轮 AI 分析脑子版本


五、首页内容与交易面板开发
1. 首页从传统介绍页逐步调整为交易内容首页
2. 核心展示逐步收口为：
   - 每日 AI 行情分析
   - 今日偏向
   - 主结论
   - 关键位置
   - 多级别指标提示
   - 开单建议
   - 宏观事件表
3. 多次调整首页信息层级与版式结构
4. 多次优化首页视觉平衡、留白、阅读路径与专业感
5. 删除冗余宣传型模块与重复 CTA
6. 保留 Discord / Telegram 联系方式入口


六、首页每日 AI 行情分析模块开发
1. 搭建每日 AI 行情分析首页主模块
2. 支持 headline / market bias / conviction / summary / key levels 等字段
3. 支持主结论、关键位置、偏向卡等模块展示
4. 多次优化首页主视觉重点与可读性
5. 后续继续向真实自动更新产品演进


七、多级别指标面板开发
1. 将右侧说明卡重构为多级别指标面板
2. 级别支持：4H / D / W
3. 指标支持：
   - VWAP
   - VAL
   - VAH
   - MACD 状态
   - MACD 背离
   - RSI 数值
   - RSI 背离
4. 按多空方向加入颜色区分
5. 调整多空强弱表达，使用红/绿深浅变化
6. 多轮优化字体、间距、行高和面板可读性


八、开单建议模块开发
1. 首页保留结构化开单建议模块
2. 支持拆分：
   - 短线
   - 长线
3. 支持展示：
   - 方向
   - 触发区
   - 止损位
   - 目标位
   - 失效条件
   - 执行备注
4. 多次调整其高亮表现、阅读密度与版式承接关系


九、宏观事件表开发
1. 首页集成 FOMC / CPI / 非农事件表
2. 支持展示：
   - 下一次时间（JST）
   - 当前 / 实际
   - 预期
   - 前值
   - 状态
   - 备注
3. 用于补充每日交易背景与风险上下文


十、首页排版与视觉精修
1. 多轮调试首页宽度、留白、模块比例
2. 优化首屏主结论与关键位置结构
3. 调整右侧指标面板密度，避免太松或太挤
4. 增加交互引导：下拉查看开单建议
5. 将引导按钮做成可点击锚点跳转
6. 调整“今日偏向”模块尺寸与情绪颜色表达
7. 调整“每日结论”在主区域中的垂直位置
8. 整体目标：更像专业交易面板，不像资讯或宣传页


十一、后台每日分析发布台开发
1. 新增后台页面：`/admin/daily-ai-market`
2. 支持手动填写每日分析完整模板
3. 支持发布为 published
4. 发布后首页可直接读取最新正式内容
5. 后台发布链路已打通


十二、正式数据源链路开发
1. 将首页逻辑从只读本地 seed 升级为优先读取正式数据源
2. 新增 Supabase 正式数据表：`daily_ai_market_analyses`
3. 支持首页优先读取最新 published 记录
4. 若无正式数据，再回退 seed
5. 使每日分析不再只是纯前端假数据


十三、自动发布接口开发
1. 新增自动发布服务层
2. 新增自动发布接口：`/api/daily-ai-market/auto-publish`
3. 支持 token 鉴权
4. 支持自动写入 Supabase
5. 写入时自动设为 published
6. 首页与 API 均可读取最新自动发布内容
7. 已支持手动接口测试成功


十四、每日自动更新定时链路开发
1. 建立每日自动发布 cron 执行方案
2. 预留 Vercel cron 配置
3. 优先接通 OpenClaw cron 使用链路
4. 配置每天 21:15（Asia/Tokyo）自动执行
5. 完成 cron 创建
6. 完成 cron 手动运行测试
7. 已形成“正式数据源 + 自动发布 + 定时任务”的闭环雏形


十五、课程学习占位开发
1. 顶部导航新增：课程学习（更新中）
2. 预留课程页路由：`/courses`
3. 当前仅为占位，不承载真实课程内容
4. 未接视频、上传、资料分发等内容能力


十六、当前 1.0 阶段开发成果总结
Twone 1.0 当前已经完成从“概念网站”到“可运行产品雏形”的核心开发工作。

本阶段真正完成的关键内容，不只是页面展示，而是以下开发链路已经接通：
1. 会员申请与审核
2. 登录与管理员后台入口
3. 首页每日分析展示
4. 后台手动发布每日分析
5. Supabase 正式数据源
6. 自动发布接口
7. 每日定时任务链路

这意味着 Twone 1.0 已经具备继续往“真正每日 AI 交易产品”方向迭代的技术基础。
