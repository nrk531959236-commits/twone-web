Twone 网站开发全流程记录

说明：
本文件用于完整记录 Twone 网站从 0 到 1.0 阶段的主要开发过程、关键决策、已完成事项、当前状态与后续方向，供后续继续开发、排错、交接和恢复上下文使用。


一、项目目标与最初方向
Twone 网站最初目标不是做普通宣传页，而是逐步构建一个围绕交易研究、会员服务和 AI 分析的产品型网站。

核心方向逐步收口为：
1. 私域会员平台
2. AI 交易助手
3. 每日 AI 行情分析
4. 会员申请 / 审核 / 开通
5. 后台管理与后续自动化发布

后续方向再进一步明确为：
- 产品重心放在 BTC 为主的交易分析
- 首页更像交易内容首页，而不是介绍页
- AI 助手需要固定入口、固定结构、固定风格
- 课程模块先预留，后续再做真实内容承载

五、AI 助手方向调整过程
AI 助手最开始更偏普通聊天入口，后面被明确要求收口为交易导向产品。

方向演变过程：
1. 从普通聊天框 → 固定交易分析入口
2. 从泛聊天 → BTC-first
3. 从自由提问 → 固定结构输出
4. 从模板初版 → 更短、更像交易员表达
5. 从只支持 BTC → 后续需要支持 BTC / ETH / SOL / BNB / 黄金

已完成的演进包括：
- AI 助手固定分析入口思路
- 权限区分（普通用户 / Pro / VIP）
- 输出风格逐步向冷静、偏进攻型交易表达靠拢
- 固定模板结构初步建立

当前仍需继续补完：
- AI 助手页最新逻辑是否完整呈现到线上
- 交易对入口补全（BTC / ETH / SOL / BNB / 黄金）
- 右侧空白区域继续补完整
- 旧按钮 / 旧发送逻辑彻底替换


六、首页从介绍页到交易内容首页的演变过程
首页经历了多轮大调整，核心方向逐步明确为：
“内容型交易首页”，不是介绍页，也不是普通 landing page。

重要调整过程：
1. 去掉大量冗长介绍内容
2. 首页核心变成每日 AI 行情分析
3. 增加今日偏向 / 主结论 / 关键位置
4. 增加开单建议
5. 增加宏观事件表（FOMC / CPI / 非农）
6. 增加多级别指标提示（4H / D / W）
7. 课程学习先做导航占位
8. Discord / Telegram 保留为联系方式入口

页面样式多轮迭代重点：
- 调整首屏比例
- 调整留白与密度
- 调整块与块之间层级
- 加入“下拉查看开单建议”交互提示
- 修偏向卡大小与颜色深浅
- 修每日结论位置
- 控制不要太空，也不要太紧凑

最开始首页每日分析是本地 seed / 假数据结构。
后续目标明确为：
- 必须有正式数据源
- 不能只是前端写个框架

已完成：
1. 新建 Supabase 表：`daily_ai_market_analyses`
2. 首页优先读取最新 `published` 内容
3. 若无正式内容，再回退 seed
4. 新增后台发布台：`/admin/daily-ai-market`
5. 可手动填写并发布当日分析

关键意义：
- 首页每日分析正式进入“可发布、可读取、可替换”状态
- 不再只是前端假结构

十一、定时任务与自动运行问题排查过程
原计划：
- 每日 21:15（Asia/Tokyo）自动执行

实际过程：
1. 已创建 cron 任务 `twone-daily-ai-market`
2. 手动运行 cron 已成功
3. 某次 21:15 没看到首页变化，后续排查发现：
   - 不是 cron 链路坏了
   - 更可能是当时网络中断导致首次自动触发没真正成功
   - 后续联网后手动补跑成功
4. Supabase 中确认存在：
   - `status=published`
   - `source=auto`
说明自动发布成功写入过

因此确定：
- 技术链路 OK
- 展示层和兜底机制需要补充


十二、首页状态文案与补跑兜底修复
针对自动更新“看起来没更新”的问题，已进行 1.0 小修复：

1. 首页状态文案不再只按时间推断
现在改为优先看数据库中当天记录是否 `published`。

2. 自动补跑机制
已设计并补入最小可用方案：
- 主跑：21:15 JST
- 兜底：22:15 JST

这意味着如果 21:15 错过或失败，22:15 再触发时可补成 `published`。

注意：
- 兜底需要第二次触发存在
- 若要真正稳定运行，应保证 22:15 也有触发来源
